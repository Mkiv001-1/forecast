## Найденные расхождения (summary)

### 1. API эндпоинты — 28 недокументированных, 5 с несоответствием пути/метода

**Недокументированные в README (существуют в api.py):**
- `GET /logs/{log_id}` — получение одного лога
- `POST /run/price-data` — обновление price data
- `GET /prompts` — список промптов
- `POST /providers` — добавление провайдера
- `DELETE /providers/{name}` — удаление провайдера
- `GET /accounts` / `POST /accounts/sync` — IB счета (пути без `/ib/`)
- `GET /portfolio` / `POST /portfolio/sync` — IB портфель
- `GET /ib/test-connection` — метод GET, а в документации POST
- `GET /ib-config`, `POST /ib-config`, `PUT /ib-config/{id}`, `DELETE /ib-config/{id}` — CRUD ib_config
- `GET /ib-order-types`, `PUT /ib-order-types/{code}/active`, `POST /ib-order-types/reset`
- `GET /trades` — список трейдов
- `GET /ib-log` — лог IB Gateway
- `POST /consensus/{id}/activate` — активация консенсуса
- `GET /consensus/{id}/preview-trade` — превью трейда
- `GET /scheduler/tasks`, `PATCH /scheduler/tasks/{name}/active`
- `GET /method-config` (list), `POST /method-config` (create), `PUT /method-config/{method}` (update)
- `GET /providers/{provider}` — получение одного провайдера
- `GET /heartbeat/history` — история heartbeat
- `GET /tickets`, `POST /tickets`, `PATCH /tickets/{id}`, `DELETE /tickets/{id}` — тикеты

**Несоответствия пути/метода:**
- Документация: `POST /ib/test-connection` → Код: `GET /ib/test-connection`
- Документация: `POST /ib/sync-accounts` → Код: `POST /accounts/sync`
- Документация: `POST /ib/sync-portfolio` → Код: `POST /portfolio/sync`
- Документация: `GET /ib/accounts` → Код: `GET /accounts`
- Документация: `GET /ib/portfolio` → Код: `GET /portfolio`

**Баг в коде:** `GET /consensus` определён дважды в api.py (строки 563 и 1153), второе определение затеняет первое.
