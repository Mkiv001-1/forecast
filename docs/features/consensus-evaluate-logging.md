# Consensus Evaluate Logging & Progress Tracking

## Описание
Улучшение видимости процесса оценки консенсусных прогнозов. Сейчас при нажатии "Evaluate Now" запускается фоновая задача без обратной связи, и непонятно:
- Сколько записей будет обработано
- Какой прогресс
- Какие ошибки возникли
- Завершилось ли успешно

## Требования

### Функциональные
1. **Системное логирование** - записи о начале/завершении evaluation в системный лог (таблица `system_log` или файл)
2. **Прогресс в GUI** - видимость сколько записей обработано / осталось
3. **Детальное логирование** - каждая обработанная запись логируется с результатом
4. **Обработка ошибок** - ошибки не должны "зависать" без уведомления

### Нефункциональные
- Минимальные изменения в архитектуре
- Сохранение асинхронной природы задачи (не блокировать GUI)

## Архитектурные решения

### Вариант 1: Простое усиленное логирование (выбрано)
Добавить детальное логирование в `consensus_evaluator.py` и `api.py`:
- Логировать начало evaluation
- Логировать каждую обработанную запись
- Логировать итоговый результат
- GUI будет показывать MessageBox с результатом после завершения (синхронный вызов с таймаутом)

### Вариант 2: Таблица task_log
Создать таблицу для отслеживания фоновых задач:
```sql
CREATE TABLE task_log (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    task_type TEXT NOT NULL,  -- 'consensus_evaluate'
    started_at TEXT NOT NULL,
    finished_at TEXT,
    status TEXT,  -- 'running', 'completed', 'error'
    records_total INTEGER,
    records_processed INTEGER,
    result_message TEXT,
    error_message TEXT
);
```

### Вариант 3: WebSocket / SSE
Потоковая передача прогресса evaluation в реальном времени.

**Выбран Вариант 1** - минимальные изменения, максимальная польза.

1. **Модифицировать `consensus_evaluator.py`**:
   - Добавить `logger.info` перед каждым `evaluate_consensus_records` с количеством найденных записей

   - Добавить `logger.info` с итоговой статистикой

2. **Модифицировать `api.py`**:
   - Endpoint `/consensus/evaluate` сделать синхронным (убрать `BackgroundTasks`)

4. **Тесты**:
   - Проверить что evaluation проходит без ошибок
   - Проверить что логи пишутся
   - Проверить что GUI получает результат
