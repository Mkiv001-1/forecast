# Кнопка форсированного пересчета всех данных

## Статус

Done

---

## Описание

Кнопка в GUI для принудительного пересчета всех данных робота — прогнозов, консенсуса, оценок.

---

## Требования

### Функциональные
- Кнопка "Recalculate All" в GUI (в панели управления или меню)
- При нажатии — запуск полного цикла пересчета
- Опционально: выбор режима (только прогнозы / прогнозы + консенсус / полный цикл с оценкой)
- Индикатор прогресса и статуса операции
- Блокировка кнопки на время выполнения

### Нефункциональные
- Асинхронное выполнение (не блокировать GUI)
- Логирование всех операций
- Обработка ошибок с уведомлением пользователя

---

## Архитектурные решения

### Компоненты
```
gui_main.py:2373-2387    Кнопка "RECALCULATE ALL" в разделе Manual Run
api.py:325-329           Существующий POST /run/full endpoint
forecast_runner.py:167-198  Полный цикл через run_trading_bot()
```

**Примечание**: Кнопка использует существующий endpoint `/run/full`, который выполняет:
1. Оценку старых прогнозов (`evaluate_past_forecasts`)
2. Генерацию новых прогнозов (`run_trading_bot`)
3. Расчет консенсуса для всех тикеров

### Поток данных
```
[GUI] Нажатие кнопки "RECALCULATE ALL"
  → [API Client] POST /run/full
  → [Server] Background task:
    - evaluate_past_forecasts(db_manager) — оценка старых прогнозов
    - run_trading_bot(db_manager) — генерация новых прогнозов
      - process_ticker() для каждого тикера
      - calculate_consensus() + save_consensus()
  → [GUI] Опрос статуса /run/status каждые 2 сек через StatusPoller

## Примечания

- Кнопка доступна на вкладке "Run/Scheduler" в разделе "Manual Run"
- Использует существующий endpoint `/run/full` (не требует изменений на сервере)
- При запуске блокируются все кнопки раздела (через `_set_run_buttons(False)`)
- После завершения кнопки разблокируются автоматически (через `_poller.finished`)
- Статус выполнения отображается в реальном времени (опрос `/run/status` каждые 2 сек)
