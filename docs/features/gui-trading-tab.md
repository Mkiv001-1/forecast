# GUI Trading Tab

## Описание
Консолидация торгового интерфейса в отдельный таб **Trading** с двумя подвкладками:
- **Trades** — просмотр сделок из таблицы `trades`
- **Orders** — просмотр/отмена ордеров из таблицы `orders`

Важно: **System Log не переносится** и остается отдельной верхнеуровневой вкладкой.

## Требования

### Функциональные
- Отдельный верхнеуровневый таб `Trading`
- Подвкладка `Trades` с фильтрами `ticker/status`
- Подвкладка `Orders` с фильтрами `ticker/status`
- Кнопка отмены выбранного ордера в `Orders`
- Двусторонняя навигация:
  - `Orders -> Trades` (по ticker + `ib_parent_id`)
  - `Trades -> Orders` (по ticker + `ib_parent_id`)
- Явные состояния:
  - пустая выборка
  - ошибка загрузки
- Режим linked-фильтрации с визуальным индикатором и быстрым сбросом

### Нефункциональные
- Без изменений backend-контрактов для MVP
- Без изменения расположения `System Log`
- Минимальные риски регрессии за счет переиспользования существующего `Orders` UI

## Архитектурные решения

### Затрагиваемые модули
- `scripts/client/gui_main.py`
  - `TradingTab` (новый контейнер)
  - `TradesTab` (новая реализация)
  - `OrdersTab` (расширен)
  - `MainWindow` (подключение Trading таба)
  - `_TabLoader` (загрузка Trading вместо старого отдельного Orders шага)
- `scripts/client/api_client.py`
  - `get_trades()` (новая обертка над `GET /trades`)

### Новый API
- Нет
- Используются существующие endpoint-ы:
  - `GET /orders`
  - `POST /orders/{id}/cancel`
  - `GET /trades`

## Реализация

### Что выполнено
1. Добавлен верхнеуровневый таб `Trading` с подвкладками `Trades` и `Orders`
2. `OrdersTab` приведен к схеме БД:
   - `Action` отображается из поля `action` (вместо `side`)
   - добавлена колонка `Role` из `order_role`
3. Добавлен `TradesTab` (read-only MVP):
   - фильтры `ticker/status`
   - таблица сделок
   - подсветка PnL (green/red)
4. Двусторонняя кросс-навигация между `Orders` и `Trades`
5. Добавлены state-label для:
   - пустых выборок
   - ошибок загрузки
6. Добавлены linked-индикаторы (`Linked mode: parent #...`) и кнопки `Reset Filters`
7. `System Log` сохранен как отдельный верхнеуровневый таб
8. `_TabLoader` переключен на загрузку `Trading` единым шагом

## Ограничения текущего MVP
- Нет polling/real-time автообновления
- Нет bulk-операций
- Нет export в CSV
- Нет отдельной формы manual order submit в GUI (доступно только через API)
